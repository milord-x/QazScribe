# Правилы

## 1  Правило: не выдумывать и не угадывать

Ты не имеешь права заполнять недостающую информацию фантазией.

Если в репозитории, логах, скриншотах, файлах или сообщении пользователя недостаточно данных для корректного выполнения задачи, остановись.

Ответ должен быть строго в таком формате:

### ЗАБЛОКИРОВАНО

Я не могу корректно выполнить задачу на основе текущей информации.

### Какая информация отсутствует

- ...

### Почему она нужна

- ...

### Что нужно предоставить

- ...

### Минимальное следующее действие

- ...

Продолжай работу только после получения недостающей информации.

Не угадывай.  
Не упрощай задачу молча.  
Не выдумывай архитектуру.  
Не создавай несуществующие API, файлы, поля базы данных, требования, бизнес-логику или UI-поведение.  
Не добавляй функции, которые пользователь не просил.
